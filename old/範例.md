# 針對多義詞 (Mars/Venus) 的建議設定範例

這份文件展示了如何在不修改程式邏輯的情況下，透過調整 `GLOSSARY` 內容與 `SYSTEM_PROMPT` 指令來解決多義詞問題。

## 1. 修改 `astrology_data.py` 中的 Gloassary

我們不再只是單純提供「詞彙 -> 譯名」的對照，而是在譯名中加入**括號提示 (Context Hints)**。LLM 看得懂這些提示，並會根據上下文選擇合適的詞。

```python
ZODIAC_GLOSSARY = {
    # ... (其他詞彙) ...

    # --- Aries (白羊座) ---
    "Aries": "白羊座",
    # [修改] 加入括號提示，讓 AI 知道何時用行星名，何時用原型名
    "Mars": "火星 (若形容性格或原型可譯為「戰士」)",
    "Warrior": "戰士",

    # --- Taurus (金牛座) ---
    "Taurus": "金牛座",
    # [修改] 加入括號提示
    "Venus": "金星 (若形容神話或原型可譯為「維納斯」)",
    
    # --- Gemini (雙子座) ---
    "Gemini": "雙子座",
    # [修改] 加入括號提示
    "Mercury": "水星 (若形容特質可譯為「信使」)",

    # ... (其他詞彙) ...
}
```

## 2. 修改 `translation_config.py` 中的 System Prompt

在 `SYSTEM_PROMPT` 中加入一條針對「歧義處理」的明確規則，教導 AI 如何解讀上述的括號提示。

```python
SYSTEM_PROMPT = f"""
Role: 你是一位專業的占星學翻譯專家...
(略...)

3. **語序優化 (核心)**：
   (略...)

# --- [新增] 第 10 點規則 ---
10. **歧義與多義詞處理 (Ambiguity Handling)**：
   - 詞彙表中若包含括號提示（如：「火星 (若...可譯為...)」），請務必根據上下文判斷。
   - **星體行運 (Transit) / 宮位 / 相位**：一律使用星體名稱（如：火星、金星）。
     - Example: "Mars in the 5th House" -> 「火星位於五宮」
   - **神話原型 / 性格特質 / 譬喻**：可靈活使用括號內的替代譯名（如：戰士、維納斯、信使）。
     - Example: "He acts like a Mars" -> 「他表現得像個戰士」
     - Example: "The beauty of Venus" -> 「維納斯的美麗」

Technical Glossary:
{GLOSSARY}

Mandatory: 請確保輸出內容完全為繁體中文...
"""
```
